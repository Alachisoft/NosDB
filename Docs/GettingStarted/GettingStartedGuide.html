Getting Started with NosDB

 

Welcome to NosDB! The following steps will get you started in ensuring smooth installation, configuration and use of NosDB.

1.     Install NosDB on Database Servers and Client Machines

 

Install NosDB on the database servers and client machines. A 64-bit database server is compatible with 32-bit clients.

2.     Open Firewall TCP Port for Clients (optional)

 

Configuration Server Port:

 

 If a firewall is enabled between the clients and the configuration server, then you must open a TCP port (default 9950) in the firewall for the client to communicate with the configuration server through the connection string.

 

"Data Source=20.200.20.44; Port=9950; Database=northwind; Local Instance=false; Integrated Security=true;"

 

If you want to manually configure the Config Server Port, changes must be made on all servers running the Configuration Service. The port can be modified in the Alachisoft.NosDB.ConfigurationService.exe.config" file located in

"%INSTALL_DIR%/bin/service":

 

  <appSettings>

    . . .

    <add key="ConfigServerPort" value="9950"/>

 

  </appSettings>

 

Management Port:

 

The configuration server further communicates with the Database Service using the Management Port. The default Management Port for the Database Service is 9960.

 

If you want to manually configure the Management Server Port, changes must be made on all servers running the Configuration Service. The port can be modified in the Alachisoft.NosDB.ConfigurationService.exe.config" file located in %INSTALL_DIR%/bin/service":

 

  <appSettings>

    . . .

    <add key="ManagementServerPort" value="9960"/>

 

  </appSettings>

 

 

However, this port is being used by the database service so it also has to be modified accordingly in the Alachisoft.NosDB.DatabaseService.exe.config" file located in "%INSTALL_DIR%/bin/service":

 

  <appSettings>

    . . .

    <add key="ManagementServerPort" value="9960"/>

 

  </appSettings>

 

 

Every time the configuration files (of Configuration Service and Database Service) are modified, the corresponding services must be restarted.

 

3.     Create New Database Cluster

 

To begin working with NosDB, the database cluster must be created through PowerShell. The New-DatabaseCluster tool creates a database cluster. However, a database cluster must contain a shard with at least one node.

 

NosDB PowerShell Provider is only compatible with PowerShell 4.0 and above.

Before creating a database cluster, the following pre-conditions must be fulfilled:

1.     All machines specified as shard nodes must have both the database and configuration services running.

2.     These machines must not be part of any other database cluster.

 

         Execute the following command into PowerShell with the parameters to create a database cluster.

 

New-DatabaseCluster Name [cluster-name] -Shard [shard-name] -Port [port] -Server [server-IP]

 

4.     Load Northwind Sample Database

 

Once the cluster has been created, you can load an existing Northwind database provided in NosDB samples at %INSTALL_DIR%\samples\data\json\northwind. The sample contains JSON format files against specific collections which can be imported into the configured database to get you started with NosDB features.

 

The provided "NorthWind.ps1" script in the same location contains all steps - creating a database, collections and importing the sample northwind data into them. The script is ready for execution:

 

   Enable PowerShell script execution by setting the ExecutionPolicy to RemoteSigned.

 

>Set-ExecutionPolicy RemoteSigned

 

   Make sure you are connected to the cluster and within the context "NosDB: \$cluster\>".

 

  Execute either of the following commands in PowerShell, based on your accessibility:

 

o     If you are accessing NosDB cluster from a remote client:

 

>Connect-DatabaseCluster Server [cluster-IP] Port [Port]

 

o    If you are accessing NosDB cluster from the local machine:

 

>Connect-DatabaseCluster

 

      Execute the following command in the context, starting with the & sign:

 

& "%INSTALL_DIR%\NoSDB\samples\data\json\NorthWind.ps1"

 

 

5.     Query Database

 

Once the data is imported into the collections it can be verified by querying or API calls. In order to query the database, execute the Invoke-SQL tool to verify the newly created database and collections by performing queries over the database.

 

The following sample queries provide a brief overview into the query language supported by NosDB and get you started with working with NosDB.

 

SELECT * FROM Products

SELECT CustomerID, OrderDate, RequiredDate, ShipCountry FROM Orders WHERE OrderID = 10248

SELECT DISTINCT SupplierID FROM Products

SELECT COUNT(OrderID) FROM Orders WHERE CustomerID ='VINET'

SELECT * FROM Products WHERE ProductName LIKE 'Ch%'

SELECT * FROM Products WHERE UnitsInStock NOT EXISTS

SELECT OrderID, CustomerID, EmployeeID FROM Orders WHERE ShipCountry IN ('Germany', 'France')

SELECT ShipName, ShipCity, ShipPostalCode FROM Orders WHERE ShipRegion ='SP' ORDER BY EmployeeID

SELECT Country, COUNT(City) AS CitiesServed FROM Suppliers GROUP BY Country

SELECT * FROM Orders WHERE ShipCity = 'London' AND Freight BETWEEN 20.0 AND 30.0

 

6.   Execute Test-Stress Tool

 

Once the database has been configured, NosDB PowerShell allows testing the stress performance of the connected database.  The Test-Stress tool creates a test collection named nosdb_test_coll and performs multiple Insert, Update, Delete and Get operations on it from single or multiple clients. This command only works in the context of the specified database.

 

PS NosDB:\cluster\databases\$database-name$> Test-Stress

 

         To kill the Test-Stress program, enter the keyboard combination Ctrl+C.

 

 

7.     Execute NosDB API Samples

 

This NosDB setup has been shipped with sample applications located at %INSTALL_DIR%\samples\data\applications to let you understand NosDB features and APIs. The samples for the following features are available:

 

    CRUDWithoutSQL Performs CRUD operations on NosDB through API.

    CRUDWithSQL Performs CRUD operations on NosDB using DDL Queries.

 

9.   Configure Visual Studio Project to Use NosDB

 

You are now ready to create .NET applications and fully utilize NosDB's features through it. However, before that, the following configurations should be made:

 

    Create a project in Visual Studio.

 

     In Solution Explorer, right-click on the application name and select Add.

 

 

     Select Reference.

 

    Reference Manager Screen appears.

 

     Select Browse from the left panel and click the Browse button to select the reference.

 

     Select the files to reference screen appears.

 

    Browse to %INSTALL_DIR%\bin\assembly\

 

    Select the following references from the list of NosDB assemblies:

        Alachisoft.NosDB.Client.dll

        Alachisoft.NosDB.Common.dll

        Alachisoft.NosDB.Core.dll

        Alachisoft.NosDB.Distributor.dll

        Alachisoft.NosDB.Serialization.dll

 

    Include the following namespaces:

        using Alachisoft.NosDB.Client;

        using Alachisoft.NosDB.Common;

        using Alachisoft.NosDB.Common.Server.Engine;

 

  You can now create your own applications, utilizing NosDBs features.
